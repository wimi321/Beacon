# セキュリティポリシー

> この文書は日本語の案内です。内容に差異がある場合は [`SECURITY.md`](./SECURITY.md) を基準にしてください。

## 対象範囲

Beacon は緊急フロー、ネイティブモバイルブリッジ、オフライン知識配信を扱います。脆弱性は責任ある形で報告してください。

## 報告方法

専用のセキュリティ窓口が整うまでは、可能なら GitHub の private security advisory を利用してください。利用できない場合は、悪用の詳細を含まない最小限の issue を作成し、非公開の連絡経路が必要であることを明記してください。

## 含めてほしい情報

- 影響を受けるプラットフォームと OS バージョン
- 端末モデル
- 再現手順
- 影響の要約
- オフライン推論、ネイティブ権限、モデル配布、知識配信のどこに影響するか

## 対応目標

- できるだけ早く受領確認を返す
- 再現して深刻度を評価する
- 詳細公開の前に修正または緩和を行う

## 対象外

- 具体的な安全・セキュリティ影響を伴わない一般的なモデル品質議論
- Beacon 固有の影響がない純粋な上流第三者問題
