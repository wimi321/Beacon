# 参与贡献 Beacon

感谢你帮助 Beacon 变得更可靠。

## 提交 PR 之前

- 先搜索已有 issue，避免重复工作。
- 尽量提交小而聚焦的 PR，而不是大范围重构。
- 如果改了 UI，最好附上截图或录屏。
- 如果修的是运行时问题，请写清设备型号、系统版本和复现步骤。

## 本地开发

```bash
npm install
npm test
dart test
```

如果你涉及移动端壳层开发：

```bash
npm run mobile:build
```

## Pull Request 检查清单

- 急救行为相关改动要尽量易审阅、易验证。
- 不要引入任何“假 AI fallback”。
- 必须保持离线优先行为，不能把关键路径变成依赖网络。
- 如果修改了 prompt 组装、会话记忆、检索或 UI 流程，请同步补充或更新测试。
- 所有用户可见文案都应尽量简单、直白、适合高压场景。

## 知识库贡献

- 优先使用权威的医疗、求生、天气和公共安全来源。
- 每个来源都要确认版权与再分发条件。
- 新来源请通过 manifest 和构建脚本接入，不要手改生成产物。

## 涉及安全的改动

如果你的改动涉及签名、模型打包、原生桥接或安全关键急救逻辑，请在 PR 里写更详细的背景和验证方式，方便重点审查。
