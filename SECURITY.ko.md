# 보안 정책

> 이 문서는 한국어 보안 안내입니다. 해석 차이가 있으면 [`SECURITY.md`](./SECURITY.md) 를 기준으로 봐 주세요.

## 범위

Beacon은 응급 워크플로, 네이티브 모바일 브리지, 오프라인 지식 전달을 다룹니다. 취약점은 책임 있게 보고해 주세요.

## 보고 방법

전용 보안 연락처가 준비되기 전까지는 가능하면 GitHub 비공개 Security Advisory를 사용해 주세요. 그것이 불가능하면 민감한 악용 세부사항을 제외한 최소한의 issue를 열고, 비공개 후속 채널이 필요하다고 적어 주세요.

## 포함해 주면 좋은 정보

- 영향을 받는 플랫폼과 OS 버전
- 기기 모델
- 재현 절차
- 영향 요약
- 오프라인 추론, 네이티브 권한, 모델 패키징, 지식 전달 중 어디에 영향을 주는지

## 대응 목표

- 가능한 빨리 접수 확인
- 재현 및 심각도 평가
- 세부 공개 전에 수정 또는 완화

## 범위 밖

- 구체적인 안전 / 보안 결함이 없는 일반적인 모델 품질 논쟁
- Beacon 고유 영향이 없는 순수 상류 제3자 문제
