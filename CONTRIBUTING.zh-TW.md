# 參與貢獻 Beacon

> 這份文件是繁體中文協作入口。若內容與英文版有歧義，請以 [`CONTRIBUTING.md`](./CONTRIBUTING.md) 為準。

感謝你幫助 Beacon 變得更可靠。

## 提交 PR 之前

- 先搜尋現有 issue，避免重複工作。
- 盡量提交小而聚焦的 PR，而不是大範圍重構。
- 如果改了 UI，請附上截圖或錄影。
- 如果修的是 runtime 問題，請寫清裝置型號、系統版本與重現步驟。

## 本地開發

```bash
npm install
npm test
dart test
```

如果你涉及行動端殼層開發：

```bash
npm run mobile:build
```

## Pull Request 檢查清單

- 與急救行為相關的改動要易於審閱與驗證。
- 不要引入任何假 AI fallback。
- 必須保持離線優先行為，不能讓關鍵路徑依賴網路。
- 如果修改 prompt 組裝、會話記憶、檢索或 UI 流程，請同步補測試。
- 所有使用者可見文案都應簡單、直接、適合高壓情境。

## 知識庫貢獻

- 優先使用權威的醫療、求生、天氣與公共安全來源。
- 每個來源都要確認授權與再散布條件。
- 新來源請透過 manifest 與建置腳本接入，不要手改生成產物。

## 涉及安全的改動

如果你的改動涉及簽名、模型打包、原生橋接或安全關鍵邏輯，請在 PR 中提供更完整的背景與驗證方式，方便重點審查。
