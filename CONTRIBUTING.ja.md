# Beacon への貢献

> この文書は日本語の協力ガイドです。内容に差異がある場合は [`CONTRIBUTING.md`](./CONTRIBUTING.md) を基準にしてください。

Beacon をより信頼できるものにするための協力に感謝します。

## PR を開く前に

- 重複作業を避けるため、既存 issue を先に確認してください。
- 大きな全面リファクタより、小さく焦点の合った PR を歓迎します。
- UI 変更にはスクリーンショットや画面録画を添えてください。
- runtime バグの場合は端末名、OS バージョン、再現手順を書いてください。

## ローカルセットアップ

```bash
npm install
npm test
dart test
```

モバイルシェルを触る場合:

```bash
npm run mobile:build
```

## Pull Request チェックリスト

- 緊急時の挙動に関わる変更は、レビューしやすく検証しやすい状態にしてください。
- 偽の AI fallback を導入しないでください。
- 重要な経路をオンライン依存に戻さず、offline-first を維持してください。
- prompt 構成、会話メモリ、検索、UI フローを変える場合はテストも追加または更新してください。
- ユーザー向け文言は簡潔で、パニック下でも理解しやすくしてください。

## 知識ベースへの貢献

- 医療、サバイバル、気象、公共安全の権威ソースを優先してください。
- すべてのソースについてライセンスと再配布条件を確認してください。
- 新しいソースは manifest と build script 経由で追加し、生成済みバンドルを手編集しないでください。

## セキュリティ感度の高い変更

署名、モデル配布、ネイティブブリッジ、安全クリティカルなガイダンスに触れる変更では、PR に背景と検証内容を詳しく書いてください。
