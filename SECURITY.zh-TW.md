# 安全策略

> 這份文件是繁體中文安全入口。若內容與英文版有差異，請以 [`SECURITY.md`](./SECURITY.md) 為準。

## 範圍

Beacon 涉及應急流程、原生行動端橋接和離線知識分發，請以負責任的方式提交安全問題。

## 回報方式

在專用安全信箱建立之前，請盡量使用 GitHub 私密 Security Advisory。如果無法使用，請先開一個不包含敏感利用細節的最小 issue，並說明你需要私密跟進渠道。

## 建議附帶資訊

- 受影響的平台與系統版本
- 裝置型號
- 重現步驟
- 風險影響摘要
- 是否影響離線推理、原生權限、模型打包或知識分發

## 響應目標

- 盡快確認收到報告
- 儘快重現並評估嚴重性
- 在公開細節前先完成修復或緩解

## 不在範圍內

- 沒有具體安全影響的模型品質爭議
- 與 Beacon 無直接關係的純上游第三方問題
