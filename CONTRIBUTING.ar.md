# المساهمة في Beacon

> هذا الملف هو دليل المساهمة بالعربية. عند وجود اختلاف في الصياغة أو المعنى، يبقى المرجع الأساسي هو [`CONTRIBUTING.md`](./CONTRIBUTING.md).

شكرا لمساعدتك في جعل Beacon أكثر موثوقية.

## قبل فتح طلب PR

- راجع القضايا الحالية أولا لتجنب العمل المكرر.
- نفضل طلبات PR الصغيرة والمركزة بدل إعادة الهيكلة الواسعة.
- أرفق لقطات شاشة أو تسجيلات عند تغيير الواجهة.
- في أخطاء runtime اذكر طراز الجهاز وإصدار النظام وخطوات إعادة الإنتاج.

## الإعداد المحلي

```bash
npm install
npm test
dart test
```

إذا كنت تعمل على الأغلفة المحمولة:

```bash
npm run mobile:build
```

## قائمة مراجعة Pull Request

- يجب أن تبقى التغييرات المرتبطة بسلوك الطوارئ سهلة للمراجعة والتحقق.
- لا تضف أي fallback ذكاء اصطناعي مزيف.
- حافظ على سلوك offline-first عندما تكون الشبكة غير متاحة.
- إذا عدلت تركيب الـ prompt أو ذاكرة الجلسة أو retrieval أو تدفق الواجهة، فحدّث الاختبارات أيضا.
- يجب أن تبقى اللغة الظاهرة للمستخدم بسيطة وواضحة ومناسبة لظروف الذعر.

## المساهمة في قاعدة المعرفة

- أعط الأولوية للمصادر الطبية ومصادر البقاء والطقس والسلامة العامة ذات الموثوقية.
- احترم شروط الترخيص وإعادة التوزيع لكل مصدر.
- أضف المصادر الجديدة عبر manifest وسكربتات البناء، وليس بتعديل الملفات المولدة يدويا.

## التغييرات الحساسة أمنيا

إذا كان التغيير يمس التوقيع أو تغليف النماذج أو الجسور الأصلية أو إرشادات الأمان الحرجة، فافتح PR مع تفاصيل وسياق واختبار أوضح.
