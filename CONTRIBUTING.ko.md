# Beacon 기여 가이드

> 이 문서는 한국어 협업 안내입니다. 해석 차이가 있으면 [`CONTRIBUTING.md`](./CONTRIBUTING.md) 를 기준으로 봐 주세요.

Beacon을 더 신뢰할 수 있게 만드는 데 도움을 주셔서 감사합니다.

## PR 열기 전에

- 중복 작업을 피하기 위해 기존 issue를 먼저 확인해 주세요.
- 큰 범위 리팩터링보다 작고 집중된 PR을 선호합니다.
- UI 변경에는 스크린샷이나 화면 녹화를 함께 올려 주세요.
- runtime 버그라면 기기 모델, OS 버전, 재현 절차를 적어 주세요.

## 로컬 설정

```bash
npm install
npm test
dart test
```

모바일 셸을 다루는 경우:

```bash
npm run mobile:build
```

## Pull Request 체크리스트

- 응급 동작과 관련된 변경은 리뷰와 검증이 쉬워야 합니다.
- 가짜 AI fallback 을 도입하지 마세요.
- 핵심 경로가 네트워크 의존으로 돌아가지 않도록 offline-first 를 유지하세요.
- prompt 조합, 세션 메모리, 검색, UI 흐름을 바꾸면 테스트도 함께 추가 또는 수정하세요.
- 사용자에게 보이는 문구는 단순하고 직관적이어야 하며, 공황 상황에서도 이해 가능해야 합니다.

## 지식베이스 기여

- 의료, 생존, 기상, 공공 안전의 권위 있는 출처를 우선하세요.
- 모든 출처의 라이선스와 재배포 조건을 확인하세요.
- 새 출처는 manifest 와 빌드 스크립트를 통해 추가하고, 생성 산출물을 직접 수정하지 마세요.

## 보안 민감 변경

서명, 모델 패키징, 네이티브 브리지, 안전 핵심 가이던스를 건드리는 변경이라면 PR에 배경과 검증 내용을 더 자세히 적어 주세요.
