# Beacon v0.1.1

Beacon `v0.1.1` 是项目公开开源后的第一轮“发布候选级”稳定性收口版本。

## 这一版主要做了什么

- 稳定了 Android 安装包“先装 App、再下模型”的轻量发布链路
- 保留真实端侧 Gemma 4 能力，不再把 GitHub 安装包做成过重的大包
- 补强了 Android / iOS 的启动路径、模型就绪引导与发布构建稳定性

## 关键产品修复

- 修复了冷启动时“只有系统提示消息也会误进聊天页”的首页状态问题
- 保证低电量告警这类系统提示不会破坏首页求生入口
- 保持 `设置与模型` 的模型就绪引导明确透明，不伪装成本地 AI 已经可用

## Android 发布收口

- 清理模板残留，收口 launcher / splash 资源，减少 lint 噪音
- 调整到与当前 R8 元数据解析更兼容的 AGP / Kotlin 组合
- 让 `mobile:build` 在 `cap sync` 之后自动做一次 Android 生成配置归一化
- 重新验证了 Debug APK、Release APK、Release AAB 的生成

## iOS 验证更新

- 重新验证了 Xcode 下的 Debug 模拟器构建与 Release 真机构建
- 重新安装并冷启动了 iOS 模拟器里的 App
- 确认冷启动后会回到求生首页，而不是落入空聊天壳

## 本轮验证命令

- `npm test`
- `npm run build`
- `npm run mobile:build`
- `dart analyze`
- `dart test`
- `cd android && ./gradlew :app:testDebugUnitTest :app:testReleaseUnitTest :app:lintDebug :app:assembleDebug :app:assembleRelease :app:bundleRelease`
- `xcodebuild -project ios/App/App.xcodeproj -scheme App -configuration Debug -sdk iphonesimulator -destination 'generic/platform=iOS Simulator' CODE_SIGNING_ALLOWED=NO build`
- `xcodebuild -project ios/App/App.xcodeproj -scheme App -configuration Release -sdk iphoneos -destination 'generic/platform=iOS' CODE_SIGNING_ALLOWED=NO build`
- `xcrun simctl install <simulator> App.app`
- `xcrun simctl launch <simulator> com.beacon.sos`
