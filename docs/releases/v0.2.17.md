# Beacon v0.2.17

Beacon `v0.2.17` adds a civilian war-crisis survival shortcut, a matching Apple-style homepage icon, and stronger offline grounding for conflict, nuclear/radiation, biohazard, chemical exposure, cyberattack, power-outage, and shelter-in-place questions.

## Highlights

- Adds a new `War Crisis` homepage panic action without removing the existing lost, earthquake, fire, trauma, visual help, or SOS flows
- Uses a cropped transparent Apple-style missile/shield icon that matches the existing homepage panic icon system
- Adds `war_crisis` as a canonical scenario so all UI languages route to the same retrieval path
- Adds `War Crisis` translations across all 20 supported UI languages
- Strengthens offline retrieval toward Ready.gov and CDC civil-defense guidance for explosions, sheltering, radiation emergencies, biohazards, hazmat, cyberattacks, public-space attacks, and outages
- Adds CDC radiation/nuclear-blast public guidance entries into the offline knowledge bundle
- Keeps DFlash out of the production mobile path after research review
- Bumps Android to `versionCode 20` / `versionName 0.2.17`

## Desktop validation

No Android phone was available for this release cycle, so this release uses desktop validation instead of real-device smoke testing.

- War-crisis icon inspected with transparent checkerboard preview
- Web UI checked locally for homepage layout, language switching, war-crisis entry, and return-home flow
- `npm test -- --run`
- `npm run build`
- `npm run competition:audit`
- `npm run mobile:android:release:github`
- Optional Android emulator smoke test if a local AVD is available

## Release asset

- `Beacon-v0.2.17-android-arm64.apk`

## Notes

The APK remains lightweight. Gemma 4 E2B/E4B model weights are downloaded from inside the app with mirror fallback and resumable download support. The war-crisis flow still uses real local model inference and offline knowledge grounding; it does not use template fallback.
