# Beacon v0.2.16

Beacon `v0.2.16` is a real-device Android polish release focused on visual rescue streaming and panic-proof SOS semantics.

## Highlights

- Routes `Visual Help / Scan Wound` image responses through the same real native `triageStream` path used by text rescue flows
- Keeps captured camera photos visible in chat while the local model is generating guidance
- Preserves the native image payload for on-device Gemma multimodal inference instead of falling back to template output
- Changes the bottom SOS control from slide/tap wording to explicit tap-only wording across all supported UI languages
- Keeps the lightweight GitHub APK path: Gemma 4 E2B/E4B model weights are still downloaded in-app
- Bumps Android to `versionCode 19` / `versionName 0.2.16`

## Real-device validation

Validated on a connected Android phone (`25019PNF3C`, `1080 x 2400`):

- APK installs successfully with `adb install -r`
- App launches without `AndroidRuntime` / `FATAL EXCEPTION`
- Home screen shows tap-only SOS copy: `点击开启 SOS 广播`
- Text panic action enters chat and streams a real local-model response
- Visual help opens the clear `Capture / Import from Album` picker
- Native camera launches successfully from Beacon
- Captured photo returns to Beacon and appears in the chat message
- Visual guidance shows `Generating advice...` before the model response appears
- SOS button toggles by tapping and changes to the stop-broadcast state

## Notes

The APK includes Beacon's offline emergency knowledge base, but it does not bundle multi-GB Gemma model weights. Users download Gemma 4 E2B or Gemma 4 E4B from inside the app, with mirror fallback and resumable download support.
