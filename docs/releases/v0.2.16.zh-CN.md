# Beacon v0.2.16

Beacon `v0.2.16` 是一版经过安卓真机验收的体验修复版，重点解决视觉求助输出方式和 SOS 底部按钮语义。

## 亮点

- `视觉求助 / 拍摄创口` 的图片回答改为复用真实原生 `triageStream` 流式通道，与文字急救问答保持一致
- 拍照后的图片会先显示在对话框里，本地模型生成期间继续显示 `正在生成建议...`
- 保留真实图片 payload 给端侧 Gemma 多模态推理，不引入模板兜底或假 AI 输出
- 底部 SOS 控件从“滑动/点击”统一改为明确的“点击”语义，所有已支持 UI 语言同步更新
- GitHub APK 继续保持轻量：Gemma 4 E2B/E4B 权重仍由用户在 App 内下载
- Android 版本升级到 `versionCode 19` / `versionName 0.2.16`

## 真机验证

已在连接的安卓手机上完成验证（`25019PNF3C`，`1080 x 2400`）：

- APK 可通过 `adb install -r` 成功安装
- App 启动无 `AndroidRuntime` / `FATAL EXCEPTION`
- 首页底部 SOS 显示 `点击开启 SOS 广播`
- 点击文字求助卡片后进入对话，并真实流式输出本地模型回答
- 视觉求助可打开清晰的 `拍摄 / 从相册导入` 面板
- 从 Beacon 可成功拉起系统相机
- 拍照确认后图片正确回到 Beacon，并显示在对话框中
- 视觉求助先显示 `正在生成建议...`，随后输出本地模型视觉分析
- SOS 按钮可通过点击切换，并进入 `停止广播` 状态

## 验证命令

- `npm test -- --run`
- `npm run build`
- `npm run mobile:android:release:github`
- Android 真机安装与烟测

## 说明

APK 内置 Beacon 离线应急知识库，但不内置数 GB 级 Gemma 模型权重。用户仍在 App 内下载 Gemma 4 E2B 或 Gemma 4 E4B，下载链路保留镜像 fallback 和断点续传能力。
