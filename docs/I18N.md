## Runtime localization flow

1. The UI exposes a manual language switcher.
2. The selected locale is persisted locally.
3. The current locale is sent through the bridge for text triage, visual help, and SOS flows.
4. Native prompt composition uses that locale so model output stays aligned with the UI language.
5. Arabic enables `dir="rtl"` for full layout reversal where required.

## Fallback rules

- Beacon does not rely only on the OS locale.
- The manually selected locale wins.
- If native text cannot be localized safely, the preferred long-term path is status codes plus frontend dictionary mapping.
- If retrieval is weak, Beacon should still answer in the selected language using real local inference.

## Contributing localization

- Keep survival and emergency phrasing short and direct.
- Avoid slang that could confuse non-native speakers.
- Preserve severity and escalation meaning.
- Verify RTL-safe layouts if touching directional UI.
- Update tests when adding or changing locale-sensitive behavior.
